# MarkStudio

## 项目简介

MarkStudio 是一款基于 Tauri 框架构建的跨平台 Markdown 编辑器，专为追求高效写作体验和团队协作的用户设计。

本项目旨在提供一个轻量、高性能的 Markdown 编辑工具，核心亮点是支持多人实时协作编辑，解决团队协作中文档同步的痛点。通过采用 Operational Transformation（OT）算法和 WebSocket 通信，实现毫秒级的编辑同步，让多人协作如同单人编辑般流畅。

**核心优势**：

- **跨平台原生体验**：基于 Tauri 2 框架，使用 Rust 编写后端，Vue 3 构建前端，提供接近原生应用的性能和体验，同时支持 Windows、macOS 和 Linux 三大平台。
- **多人实时协作**：内置完整的协作功能，支持创建/加入协作房间、实时同步编辑内容、共享多个文件。
- **智能编码处理**：自动检测文件编码（UTF-8、GBK、GB18030），完美兼容中文 Windows 环境。
- **丰富的编辑功能**：支持实时预览、大纲导航、多标签页管理、图片粘贴、主题切换等实用功能。
- **数据持久化**：使用 SQLite 本地存储，保存用户设置、最近文件、收藏夹和标签页状态。

**适用场景**：

- 个人技术文档编写和笔记记录
- 团队协作编写技术文档和 README
- 远程协作场景下的文档同步编辑

## 功能特性

### 核心编辑功能

- **Markdown 编辑**：支持完整的 Markdown 语法编辑
- **实时预览**：三种编辑模式：源代码模式、预览模式、双屏分屏模式
- **大纲导航**：自动提取文档标题结构，支持快速跳转
- **多标签页管理**：支持同时打开多个文档，独立管理每个标签页的状态

### 文件管理

- **最近文件列表**：自动记录最近打开的文件，方便快速访问
- **收藏夹功能**：支持创建目录分类收藏常用文件
- **多编码支持**：自动检测文件编码（UTF-8、GBK、GB18030），兼容中文 Windows 环境
- **标签页恢复**：启动时可自动恢复上次关闭前的标签页状态

### 多人实时协作

- **房间创建/加入**：支持创建协作房间或加入他人房间
- **实时同步**：基于 WebSocket 实现文档内容的实时同步
- **OT 算法**：采用 Operational Transformation 算法解决多人并发编辑冲突
- **光标同步**：实时显示协作者的光标位置
- **图片同步**：支持协作过程中同步粘贴的图片
- **共享文件**：主机可共享多个文件供协作者编辑

### 其他功能

- **图片粘贴**：支持从剪贴板粘贴图片，自动保存到缓存目录
- **主题切换**：支持浅色/深色主题
- **快捷键**：Ctrl+S 保存、Ctrl+F 查找
- **设置管理**：可自定义图片缓存目录、标签页恢复等选项

### 前端架构

前端采用 Vue 3 组合式 API，组件化设计，职责分明：

- **编辑器组件** (`src/components/Editor.vue`)：Markdown 文本编辑，支持协作模式下的光标显示和操作发送
- **预览组件** (`src/components/Preview.vue`)：解析后的 HTML 渲染，支持 KaTeX 公式和 Mermaid 图表
- **协作面板** (`src/components/CollaborationPanel.vue`)：协作房间管理，创建/加入房间，共享文件管理
- **侧边栏组件** (`src/components/SidebarTabs.vue`)：大纲导航、最近文件、收藏夹三个选项卡
- **状态管理**：通过 `App.vue` 集中管理多标签页、协作状态、主题等全局状态
- **组合式函数** (`src/composables/`)：`useTheme` 管理主题切换，`useShortcuts` 注册快捷键

### 后端架构

后端使用 Rust 语言，基于 Tauri 2 框架构建，分为以下模块：

- **命令层** (`src-tauri/src/commands.rs`)：定义所有 Tauri IPC 命令，作为前端与后端的通信桥梁
- **解析层** (`src-tauri/src/parser.rs`)：使用 pulldown-cmark 解析 Markdown，生成 HTML 和大纲
- **服务层** (`src-tauri/src/services/`)：核心业务逻辑，包含文件服务、图片服务、文档服务、设置服务和 OT 服务
- **数据库层** (`src-tauri/src/database.rs`)：SQLite 数据库操作，使用单例模式和 Mutex 保证线程安全
- **协作层** (`src-tauri/src/collaboration/`)：多人协作功能，包含以下子模块：
  - `ot.rs`: OT 算法实现，解决并发编辑冲突
  - `network.rs`: WebSocket 消息协议定义
  - `session.rs`: 房间会话管理，处理创建/加入/离开
  - `sync.rs`: 图片等二进制文件的分片传输

### 协作架构

协作功能采用客户端-服务器模式，基于 WebSocket 实现实时通信：

```
┌──────────┐          WebSocket          ┌──────────┐
│  Host    │ ◄────────────────────────►  │ Client 1 │
│ (主机)   │                              │  (客户端) │
├──────────┤                              ├──────────┤
│ - 房间管理 │                              │ - 加入房间 │
│ - OT 变换  │                              │ - 同步编辑 │
│ - 文件同步  │                              │ - 图片接收 │
└────┬─────┘                              └────┬─────┘
     │                                         │
     │                    ┌──────────┐          │
     │                    │ Client 2 │          │
     └──────────────────►│  (客户端) │◄─────────┘
                         └──────────┘
```

**OT 算法工作流程**：

1. 用户在编辑器中进行编辑操作
2. 前端计算编辑前后的差异，生成 OT 操作（Insert/Delete）
3. OT 操作通过 WebSocket 发送给其他协作者
4. 接收方使用 OT 变换算法处理冲突，确保最终文档一致性
5. 应用变换后的操作到本地文档，实现实时同步

### 数据持久化

项目使用 SQLite 作为本地数据库，包含以下表结构：

- **settings**：键值对形式存储用户配置（主题、图片缓存目录、标签页恢复等）
- **recent_files**：最近打开的文件列表，包含路径和打开时间
- **favorites**：收藏夹目录结构和收藏的文件
- **open_tabs**：上次关闭时的标签页状态，用于启动时恢复

数据库文件位于项目根目录下的 `data/markstudio.db`。

## 快速开始

### 环境要求

- Node.js 18+
- Rust 1.70+
- 系统依赖（参考 [Tauri 官方文档](https://tauri.app/start/prerequisites/))

### 安装依赖

```bash
# 安装前端依赖
npm install

# Rust 依赖会在构建时自动安装
```

### 开发模式

```bash
# 启动开发服务器（前端热更新 + 后端自动重编译）
npm run tauri dev
```

### 构建发布

```bash
# 构建生产版本
npm run tauri build
```

构建产物位于 `src-tauri/target/release/bundle/` 目录下，支持：
- Windows：`.msi`、`.exe`
- macOS：`.dmg`、`.app`
- Linux：`.deb`、`.AppImage`

## 协作功能使用说明

### 创建房间

1. 点击工具栏的"协作"按钮打开协作面板
2. 输入用户名和端口（可选密码）
3. 点击"创建房间"
4. 将显示的 IP 地址和房间 ID 分享给协作者

### 加入房间

1. 打开协作面板
2. 输入主机 IP、端口、房间 ID 和用户名
3. 如有密码保护，输入密码
4. 点击"加入房间"

### 协作编辑

- 所有编辑操作会实时同步给房间内的其他用户
- 可以看到其他用户的光标位置
- 主机可以共享多个文件供协作者编辑
